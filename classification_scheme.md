# 基于会话内容的三层分类方案

## 分类标准

### 第1层：高置信度大促相关

**定义**: 用户消息中明确提到大促、价格优惠、免费活动等

**判断规则**:
1. 明确提到大促关键词
   - 522、5.22、5月22
   - mega sale、flash sale、anniversary sale
   - free day、free sale、free order
   - 大促、周年庆、活动

2. 免费活动相关
   - 包含"free"/"免费"/"0 rs"
   - 且同时提到"sale"/"order"/"cloth"/"item"

3. 价格优惠咨询
   - discount、offer、deal、promotion
   - coupon、voucher、promo code
   - 折扣、优惠券、促销

4. 技术问题（大促期间特有）
   - crash、not working、error、network unavailable
   - 且同时提到时间："8:00"、"9:00"、"waited"、"since"
   - 表明用户专门等待大促开始

**细分类型**:
- 明确提到大促
- 免费活动
- 价格优惠咨询
- 技术问题-大促期间

---

### 第2层：不确定

**定义**: 无法从消息内容判断是否因大促引发

**判断规则**:
1. 订单取消但未说明原因
   - 只说"cancel"但不说为什么

2. 配送查询但未说明是否因大促
   - 只问"delivery"/"shipping"/"tracking"

3. 退款退货但未说明原因
   - 只说"refund"/"return"

4. 只发送订单卡片（JSON格式），无文字说明
   - 无法判断用户意图

5. 其他无法判断的情况

**细分类型**:
- 订单取消-原因不明
- 配送查询-原因不明
- 退款退货-原因不明
- 发送订单卡片-无文字说明
- 其他不确定

---

### 第3层：高置信度非大促

**定义**: 明确的日常问题，与大促无关

**判断规则**:
1. 简单问候/寒暄
   - hello、hi、hey、thank you、how are you
   - 内容短（<50字符）

2. 账户/会员问题
   - password、login、account、email、phone number
   - 账户、密码

3. 商品咨询（非价格相关）
   - size、material、fabric、color、measurement
   - 尺码、材质、颜色
   - 不包含价格相关词汇

**细分类型**:
- 问候/寒暄
- 账户问题
- 商品咨询

---

## 分类逻辑流程图

```
开始
 ↓
检查是否包含"第1层"任一关键词
 ├─ 是 → 第1层：高置信度大促相关
 └─ 否 ↓
    检查是否包含"第3层"任一关键词
     ├─ 是 → 第3层：高置信度非大促
     └─ 否 → 第2层：不确定
```

---

## 预期分布（基于100条样本测试）

- 第1层（高置信度大促相关）：约40-50%
- 第2层（不确定）：约35-45%
- 第3层（高置信度非大促）：约5-15%

---

## 与AI标签方法的区别

| 维度 | AI标签方法 | 消息内容方法 |
|------|-----------|-------------|
| 数据源 | AI自动标注的标签 | 用户实际发送的消息 |
| 覆盖率 | 88%（04-05时段） | 100% |
| 准确性 | 依赖AI质量 | 直接判断 |
| 分类依据 | AI理解的意图 | 明确的关键词 |
| 不确定性处理 | 归入某一类 | 单独标注"不确定" |

---

## 优势

1. **透明可验证**: 任何人看到消息都能判断分类是否正确
2. **不依赖AI**: 不受AI标注质量影响
3. **诚实标注不确定性**: 对无法判断的情况明确标注
4. **可调整**: 随时可以调整关键词规则

## 局限

1. **关键词局限**: 只能识别明确包含关键词的会话
2. **语义理解弱**: 无法理解隐含意图
3. **多语言**: 需要维护多语言关键词库
